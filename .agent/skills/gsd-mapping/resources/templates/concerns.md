# Codebase Concerns Template
Template for `.planning/codebase/CONCERNS.md` - captures known issues and areas requiring care.

## File Template
```markdown
# Codebase Concerns

**Analysis Date:** [YYYY-MM-DD]

## Tech Debt
- [Area]: [Issue/Impact/Fix]

## Known Bugs
- [Description]: [Symptoms/Root Cause]

## Security Considerations
- [Area]: [Risk/Recommendation]

## Performance Bottlenecks
- [Operation]: [Problem/Measurement/Cause]

## Fragile Areas
- [Component]: [Why/Safe Modification]

## Scaling Limits
- [Resource]: [Capacity/Limit]

## Dependencies at Risk
- [Package]: [Risk/Impact]

## Missing Critical Features
- [Gap]: [Problem/Impact]

## Test Coverage Gaps
- [Area]: [What's missing/Risk]
```
